# Linux‑Pentesting – Methodische Notizen

Dieser Ordner enthält strukturierte Notizen und Gedankengerüste zum Pentesting von Linux‑Systemen.

Die Inhalte sind phasenbasiert aufgebaut und orientieren sich an realistischen Angriffspfaden in Linux‑Umgebungen.  
Der Fokus liegt auf dem methodischen Verständnis von Zusammenhängen, Entscheidungswegen und typischen Zugriffsmöglichkeiten.

Abgebildet wird die vollständige Kill Chain:
Reconnaissance, Initial Access, Execution, Privilege Escalation, Credential Access, Lateral Movement, Persistence und Cleanup.

Die Inhalte stellen **keine Schritt‑für‑Schritt‑Anleitung** dar, sondern dienen als Lern‑ und Denkstruktur.
