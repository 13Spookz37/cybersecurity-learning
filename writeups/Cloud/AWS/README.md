Dieses Diagramm zeigt ein phasenbasiertes, methodisches Vorgehensmodell

Der Fokus liegt auf logischen Zusammenhängen, Abhängigkeiten und möglichen

oder einzelnen Labs.

Das Repository sammelt reflektierte Writeups und strukturierte Notizen zu
AWS‑Szenarien (u. a. CloudGoat). Ziel ist es, ein mentales Modell für typische
Angriffsflüsse in der Cloud zu entwickeln.

Die Inhalte dienen als methodische Referenz und Denkstruktur – nicht als
Schritt‑für‑Schritt‑Anleitung oder Spoiler.
